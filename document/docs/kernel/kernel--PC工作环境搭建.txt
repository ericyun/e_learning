4.3.     建立HISI开发环境
4.3.1.     Hi3535 SDK安装
1、后续操作需要root用户权限
   在"Hi3535_V100R001***/01.software/board"目录下，您可以看到一个 Hi3535_SDK_Vx.x.x.x.tgz 的文件，该文件就是Hi3535的软件开发包。
2、解压缩SDK包
   在linux服务器上（或者一台装有linux的PC上，主流的linux发行版本均可以），使用命令：tar -zxf Hi3535_SDK_Vx.x.x.x.tgz ，解压缩该文件，可以得到一个Hi3535_SDK_Vx.x.x.x目录。
3、展开SDK包内容
   返回Hi3535_SDK_Vx.x.x.x目录，运行./sdk.unpack(请用root或sudo权限执行)将会展开SDK包打包压缩存放的内容，请按照提示完成操作。如果您需要通过WINDOWS操作系统中转拷贝SDK包，请先运行./sdk.cleanup，收起SDK包的内容，拷贝到新的目录后再展开。
4、在linux服务器上安装交叉编译器
    1）安装uclibc交叉编译器（注意，需要有sudo权限或者root权限）：
进入Hi3535_SDK_Vx.x.x.x/osdrv/toolchain/arm-hisiv100nptl-linux目录，运行chmod +x cross.install，然后运行./cross.install即可。
    2）安装glibc交叉编译器（注意，需要有sudo权限或者root权限，暂时不支持glibc版本）：
进入Hi3535_SDK_Vx.x.x.x/osdrv/toolchain/arm-hisiv200-linux目录，运行chmod +x cross.install，然后运行./cross.install即可。
    3）执行source /etc/profile， 安装交叉编译器的脚本配置的环境变量就可以生效了，或者请重新登陆也可。

5.  按照下面过程建立HISI环境，最好能够做成脚本来执行。
mnt_qt.tar解压到 /mnt/qt下，这是QT的编译环境
home_eric_hisi.tar 解压到 /home/eric下；home_karry_hisi3535.tar解压到 /home下
usr_local_siplib.tar解压到/usr/local 目录下。

genext2fs mkimage copy到可路径 /usr/bin下
6、编译salve模式的osdrv
     source /etc/profile
     cd /home/eric/Hi3535_SDK_Slave/osdrv
     make OSDRV_CROSS=arm-hisiv200-linux CHIP=hi3535 RLS_VERSION=release PCI_MODE=slave all
7. 编译产生新的image
     cd /home/eric/Hi3535_SDK_Slave/osdrv/pub/rootfs_glibc
     ./mkimg.rootfs . .. test cramfs-initrd
     mv ../test.cramfs.initrd ../cramfs.initrd.img

/etc/profile文件中添加：
export QMAKESPEC="/mnt/qt/hisi_directfb/mkspecs/qws/linux-arm-hisi-g++"
export PATH="$PATH: /opt/hisi-linux/x86-arm/arm-hisiv200-linux/target/bin"
建立新的虚拟机，发现qmake无法正常工作，增加 export QMAKESPEC="qws/linux-arm-hisi-g++"，然后就会加载适合的qmake.conf

QT，为了避免重复编译MOC文件，应该减少头文件的重复包含。
